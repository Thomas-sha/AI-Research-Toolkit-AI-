# 战略前瞻与情景规划

> 22年研究经验 × AI加速 × 未来学方法论
>
> 核心命题：从散点趋势中提炼结构性判断，帮客户回答"未来可能走向哪里"和"我现在该押注什么"。

---

## 一、方法论核心

### 1.1 三个阶段

```
阶段1                阶段2                  阶段3
趋势扫描              张力分析                情景构建
(What's changing)     (What tensions)         (What scenarios)
   │                     │                      │
   ▼                     ▼                      ▼
PESTEL扫描结果      →  4对对立张力         →  4象限情景
20-30个趋势信号         ↓                     每个情景：
                       未来三角归纳             产业特征+关键信号+战略启示
                       (拉力/推力/权重)
```

### 1.2 与传统方法的差异

| 维度 | 传统战略咨询 | AI增强方法 |
|------|------|------|
| 趋势收集 | 人工阅读+标注，1-2周 | AI深度扫描3-4小时 + 人工审核2-3天 |
| 张力提取 | 资深顾问凭经验归纳 | AI辅助识别对立信号 + 人工筛选校准 |
| 情景构建 | 顾问主导，主观性强 | 结构化方法论（未来三角），可复现 |
| 效率 | 全流程3-4周 | 全流程1.5-2周 |

### 1.3 AI在这个方法论中的角色边界

```
AI能做：                         人必须做：
├─ 从海量信息中提取趋势信号        ├─ 选择哪两个不确定性维度做轴
├─ 识别信息中的对立观点            ├─ 判断趋势是信号还是噪音
├─ 生成情景描述的初稿              ├─ 决定报告的故事线和叙事角度
├─ 文生图辅助可视化                ├─ 对客户的战略建议——该押注什么
└─ 提供候选维度                   └─ 数据准确性最终责任
```

## 二、Phase 1：趋势信号 → 张力提取

### 2.1 趋势信号提取与评分

**输入**：案头研究PESTEL扫描结果（参考 `desk-research-ai` Skill）

**方法**（AI辅助）：

1. 将Phase 1扫描结果输入AI
2. 要求AI提取20-30个具体趋势信号——要求：可验证、有明确证据、非泛泛而谈
3. 人工对每个信号做双维评分：

```
信号强度（1-5）：证据充分程度 + 出现频率 + 信源可信度
影响程度（1-5）：对产业的潜在影响 + 波及范围

交叉矩阵：

影响程度
  ↑
5 │  黑天鹅区              核心趋势区
  │  (低信号-高影响)         (高信号-高影响)
  │  → 需持续监测            → 优先投入研究
  │
3 │  噪音区                渐显趋势区
  │  (低信号-低影响)         (高信号-低影响)
  │  → 暂时搁置             → 观察窗口
  │
1 └──────────────────────────→ 信号强度
  1         3          5
```

**人工核心判断**：
- 信源可信度加权——AI不区分信源质量，人工注入三级信源权重
- 行业specific的校准——哪些信号在别的行业是噪音，在这个行业是黑天鹅？只有行业老兵知道

### 2.2 趋势张力提取

**什么是趋势张力**：一对方向相反、当下无法确定胜负、但对未来产业形态有决定性影响的力量。不是"趋势A vs 趋势B"，而是"同一趋势的两个极端可能走向"。

**方法**（AI辅助）：

1. 将2.1的信号矩阵输入AI
2. 要求AI识别"存在明显分歧或对立信号的领域"
3. 人工从AI候选清单中筛选——三条标准：
   - **可辩论性**：正反双方都有合理论据，不是单边压倒
   - **战略相关性**：押注哪一极会导致实质上不同的资源投入
   - **时效性**：在客户的规划周期内（通常3-7年）会发生实质性变化

**数量控制**：4对。太多失去聚焦，太少覆盖不足。

### 2.3 实操示例（脱敏 · 汽车行业）

| 张力对 | 一极 | 另一极 |
|------|------|------|
| 张力1 | 品牌向上——用户愿意为品牌溢价付费 | 性价比回归——用户更关注实用价值和价格 |
| 张力2 | 技术激进——L4快速落地，用户期待革命性体验 | 用户保守——用户对新技术信任不足，偏好渐进改善 |
| 张力3 | 个性化——千人千面的产品定义和用户旅程 | 平台化——标准化硬件降本增效，个性化仅在软件层 |
| 张力4 | 制造驱动——核心竞争力仍是制造效率和供应链 | 科技驱动——核心竞争力转向软件、数据和AI |

## 三、Phase 2：未来三角归纳 → 4象限情景

### 3.1 未来三角（Futures Triangle）

```
          ┌─────────────────┐
          │   未来产业形态    │
          │  （4个象限情景）   │
          └────────┬────────┘
                   │
     ┌─────────────┼─────────────┐
     │             │             │
  ┌──▼──┐     ┌───▼───┐     ┌──▼──┐
  │拉力  │     │ 推力   │     │权重  │
  │(愿景) │     │(趋势)  │     │(惯性) │
  └─────┘     └───────┘     └─────┘
```

| 力 | 解释 | 来源 | 在项目中怎么用 |
|------|------|------|------|
| **拉力（Pull）** | 未来愿景——产业/用户/社会希望走向哪里 | 消费者期望、政策愿景、技术理想 | 工作坊中激发"如果我们能做到" |
| **推力（Push）** | 当前趋势——技术和市场正在推动什么 | PESTEL扫描结果、趋势信号矩阵 | 提供"已经不可逆转的改变"的证据 |
| **权重（Weight）** | 历史惯性——什么在阻碍改变 | 产业路径依赖、制度惯性、基础设施锁定 | 检验"愿景中短期实现可能性" |

### 3.2 不确定性维度的选择（全项目最重要的判断）

**方法**：

1. 将4对张力向上抽象，合并为2个更高层级的"不确定性维度"
2. 两大维度正交 = 4象限情景矩阵
3. **维度选择标准**：对客户战略选择有最大区分度——不同象限需要客户做出不同的资源承诺

**这是AI完全做不了的一步。** 原因：需要同时理解——
- 客户的战略优先级
- 产业的技术路径
- 组织的资源和能力边界
- 客户决策层的认知偏好

AI能列出10个候选维度，但选哪个——靠的是20年+的行业判断。

### 3.3 4象限情景构建

**每个象限的定义模板**：

```markdown
## 情景X：[名称]

### 驱动条件
- 维度1走向：[描述]
- 维度2走向：[描述]

### 该情景下的产业特征
- 产品形态
- 竞争格局
- 用户关系
- 价值链分布

### 关键信号
- 已出现的信号（支持该情景正在发生的证据）
- 还缺什么信号（如果这个信号出现，该情景概率大幅上升）

### 对客户的启示
- 如果这个情景成真，你需要在哪方面建立能力？
- 哪些现有能力会贬值？
```

### 3.4 可视化

**文生图辅助**：
- 每个象限生成一张概念图（Nano Banana / 豆包 / Midjourney）
- 用途：工作坊刺激物 + 报告感染力（非核心，锦上添花）
- ⚠️ AI生图构图和细节不可控，需迭代2-3轮调整prompt

## 四、Phase 3：从情景到需求

### 4.1 定性深访（验证情景假设）

**"分人归纳 + 分城合并 + AI框架"三步法**：

```
Step A：分人归纳（AI辅助）
  每位受访者 → AI提取15-20条关键发现
  → 人工逐条审核（AI在定性分析中幻觉率高）
  → 每人一页"用户洞察卡"

Step B：分层合并（人工）
  按城市/人群分层，横向对比同层共性需求和差异
  AI辅助寻找跨人群模式

Step C：全局整合（AI搭框架 + 人工填充）
  AI拟定全局分析框架
  人工将Step A+B的结构化信息填充
  → 定性分析完整报告
```

> ⚠️ 为什么不能跳过Step A和B，直接让AI分析20-30万字原始笔录？因为直接全局AI分析 = 幻觉放大器。分人→分层→全局的三段式确保每一步都有研究员把控点。

### 4.2 定量验证

- AI辅助：SPSS syntax自动生成（交叉表 + 显著性检验）
- **AI不介入**：数据解读（定量错误更难发现，全部人工）

## 五、全流程工具链

```
Phase 1: Deep Search(秘塔/Gemini/Perplexity) → PPT生成(天工/Gamma/Beautiful.ai)
Phase 2: DeepSeek/Claude(张力分析+未来三角) → 文生图(Nano Banana/豆包/Midjourney)
Phase 3: Claude(定性框架+分人归纳) + SPSS syntax AI辅助
Phase 4: Claude("以终为始"报告规划+定性初稿框架)
全流程: 人工(数据核查+图表重制+去AI味+方向定义+不确定性维度选择)
```

## 七、人工判断的三个不可替代节点

### 7.1 不确定性维度的选择
全项目最重要的判断。AI能列候选，不能做选择。

### 7.2 数据准确性的终身责任制
研究员对报告中每一个数据负有最终责任。AI可以辅助，不能分担。

### 7.3 故事线的叙事判断
同样的数据，不同的叙事 → 完全不同的客户反应。报告是让客户"被验证"还是"被挑战"？结论点是一个答案还是两个选项？20年+经验的终极体现——不是分析能力，是对决策者的理解。

## 九、常见风险

| 风险 | 严重度 | 应对 |
|------|:--:|------|
| 选错不确定性维度 | 🔴致命 | 做之前和客户对齐战略优先级。不要猜。 |
| AI幻觉导致报告中出现假数据 | 🔴致命 | 逐条反向验证。递交前执行最终审核。 |
| 张力提取过于抽象，缺乏战略指导性 | 🟡严重 | 每对张力必须回答"押注不同极会导致什么具体行动差异" |
| AI过度概括定性发现 | 🟡严重 | 严格遵守三步法。绝不让AI直接分析原始笔录。 |
| 情景构建偏向研究员偏好的方向 | 🟡中等 | 每个象限要给同等权重的情景推演，不预设偏好 |
